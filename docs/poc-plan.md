# 端到端 PoC 计划与任务拆分

## 目标
- 验证客户端提交打印作业，服务端接收并打印，状态回传。

## 里程碑
### M1：服务端打印 API
- 建立 Rust HTTP 服务
- 支持 /printers 与 /jobs
- 在 Windows 上调用 Win32 打印接口

### M2：客户端 CLI
- CLI 支持指定服务端与打印机
- 上传 PDF/RAW
- 查询作业状态

### M3：双面打印链路
- 服务端读取能力
- 客户端配置 duplex
- 作业打印验证

### M4：虚拟打印机 PoC
- 初期用方案 C：输出到文件夹 + 客户端上传
- 保证用户可通过“打印”操作触发远程打印

## 任务拆分
- 服务器
  - 打印机枚举与能力读取
  - 打印作业创建与内容上传
  - 作业队列与状态查询
- 客户端
  - 配置文件管理
  - 作业上传与状态查询
  - 虚拟打印机拦截/输出采集
- 协议
  - API 规范
  - 错误码与日志规范

## 风险与验证
- 双面参数在不同驱动中的一致性
- 不同应用输出格式兼容性
- 大文件传输稳定性

## 验收标准
- 一次配置后客户端可打印
- 支持双面打印
- 作业状态正确回传
